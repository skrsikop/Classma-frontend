# Contributing

Thank you for your interest in contributing to this project. The project follows a lightweight, standard workflow to make contributions simple and predictable.

Guidelines
- Fork the repository and create a feature branch from `main`.
- Keep changes focused and smallâ€”one feature or fix per PR.
- Write clear commit messages and provide a concise PR description.
- Ensure TypeScript compiles cleanly and run linting where applicable.

Pull Request Checklist
- New code compiles and runs locally.
- Add or update tests where appropriate.
- Update documentation (README, env examples) when adding features or config.

Development Workflow
1. Fork -> Create branch: `git checkout -b feat/your-feature`
2. Implement changes and commit.
3. Push your branch and open a Pull Request against `main`.

Code of Conduct
Please be respectful and follow inclusive, professional behavior when interacting in issues and pull requests.
